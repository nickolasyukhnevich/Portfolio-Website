Case Study

Doctor–Patient Database Website

Transforming fragmented patient records into a single, structured digital experience.

This project replaces paper folders and scattered spreadsheets with a web-based patient database. The goal was to reduce cognitive load for the doctor, speed up appointment prep, and make key information easier to find during a consultation.

Project Details

  • Role: UX/UI Designer, Web Developer
  • Tools: Figma, HTML, CSS, JavaScript
  • Focus: Information architecture, clean visual hierarchy

Final Interface

High-fidelity layout of the main patient dashboard.

Process & Progress

Three key stages from early structure to refined visual design.

01 — Low-Fidelity Wireframe

I started with a simple wireframe outlining the placement of navigation, the patient list, and individual patient details. My priority at this phase was clarity of structure—testing how information would flow and how users would interact with the system.

02 — Structure & Grouping

Next, I grouped related fields into clear sections: patient overview, medical history, and visit notes. This reduced scrolling and made it easier for the doctor to scan during an appointment.

03 — Visual Polish

Finally, I refined typography, spacing, and color accents for key data. The result is a calmer interface where important details stand out without feeling overwhelming.
